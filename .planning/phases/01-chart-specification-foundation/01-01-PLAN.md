**Step 1 — Extend lib/db/schema.ts:**

Add `columns: text("columns").array()` to the sessions table. Place it after `columnCount`:

```typescript
export const sessions = pgTable("csv_sessions", {
  id: varchar("id", { length: 36 }).primaryKey(),
  userId: text("user_id").notNull(),
  fileName: text("file_name").notNull(),
  rowCount: integer("row_count").notNull().default(0),
  columnCount: integer("column_count").notNull().default(0),
  columns: text("columns").array(),           // NEW — nullable, string[] | null
  sizeBytes: integer("size_bytes").notNull().default(0),
  ...timestamps,
});
```

The column is nullable (no `.notNull()`) so existing rows are backward compatible.

**Step 2 — Generate Drizzle migration:**

Run: `npx drizzle-kit generate`

This produces a new migration file in `drizzle/` (e.g., `0005_add_sessions_columns.sql` — number
will be auto-assigned by drizzle-kit). The migration SQL should be:
```sql
ALTER TABLE "csv_sessions" ADD COLUMN "columns" text[];
```

Verify the generated SQL matches the above before proceeding.

**Step 3 — Apply the migration:**

Run: `npx drizzle-kit migrate`

If `drizzle-kit migrate` is not configured, run: `npx drizzle-kit push` (pushes schema changes
directly to the Neon DB — acceptable for development).

**Step 4 — Update app/actions/upload.ts:**

In the `db.transaction` block (lines 82-99), add `columns: meta.columns` to the
`tx.insert(sessions).values({...})` call:

```typescript

  rowCount:    meta.rowCount,
  columnCount: meta.columnCount,
  columns:     meta.columns,       // NEW — persists column name array
  sizeBytes:   file.size,
});
```

meta.columns is already computed at line 74 in `chunkRows(data, meta.columns, 50)`.
No additional parsing required.

**Step 5 — TypeScript check:**

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | upload.ts columns insert | mitigate | Column names come from server-side PapaParse of the uploaded file bytes — not from client-supplied form fields. | T-01-02 | Denial of Service | recharts install | accept | recharts@2.15.4 is a pinned, audited version from npm. No runtime server exposure — it is a client-side rendering library. |
| T-01-03 | Elevation of Privilege | csv_sessions.columns field | accept | The new `columns` field stores CSV header names (non-sensitive). Access requires an authenticated session — existing auth check in uploadCSVAction guards the insert. |
</threat_model>
